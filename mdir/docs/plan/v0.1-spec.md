| 실행 파일 | GREEN | mode & S_IXUSR |

### Linux 적응

- DOS의 8.3 파일명 제한 없음 → 긴 파일명 표시
- 확장자가 없는 파일도 있음
- 단일 열 또는 복수 열 선택 가능

### 열 레이아웃 옵션

```
[단일 열 - 상세 정보]

filename.ext     another.txt    third.doc       fourth.cpp
```

v0.1에서는 **단일 열 상세 정보** 모드로 시작.

## 키 입력 처리

### Escape Sequence 파싱

```cpp
// 화살표 키
ESC [ A  → Up
ESC [ B  → Down
ESC [ C  → Right
ESC [ D  → Left

// 기능키
ESC O P  → F1 (VT100 모드)
ESC [ 1 1 ~  → F1 (xterm 모드)
ESC [ 1 2 ~  → F2
...
ESC [ 2 4 ~  → F12

// 수정자
ESC [ 1 ; 2 A  → Shift+Up
ESC [ 1 ; 3 A  → Alt+Up
ESC [ 1 ; 5 A  → Ctrl+Up

// Home/End
ESC [ H  → Home
ESC [ F  → End
ESC [ 1 ~  → Home (일부 터미널)
ESC [ 4 ~  → End (일부 터미널)

// Page Up/Down
ESC [ 5 ~  → PageUp
ESC [ 6 ~  → PageDown
```

### 입력 타임아웃

- ESC 단독 입력과 시퀀스 시작을 구분하기 위해 타임아웃 필요
- 일반적으로 50-100ms

## 파일 시스템 인터페이스

struct FileEntry
{
    std::string name;
    std::filesystem::file_type type;
    std::uintmax_t size;
    std::filesystem::file_time_type modTime;
    std::filesystem::perms permissions;

    bool isHidden;

{
public:
    std::vector<FileEntry> readDirectory(std::filesystem::path const & path);
    bool changeDirectory(std::filesystem::path const & path);
    std::filesystem::path currentPath() const;
    std::filesystem::space_info spaceInfo() const;

    bool execute(std::filesystem::path const & file);
};

## 구현 순서

### Phase 1: vtty 기반

1. `Terminal` 클래스 - raw mode 제어
2. `Screen` 클래스 - 화면 버퍼 및 출력
3. `Input` 클래스 - 키 입력 파싱
4. 테스트: 기본 출력 및 입력 확인

### Phase 2: 기본 위젯

1. `Widget` 기본 클래스
2. `Box` - 테두리 그리기
3. `Label` - 텍스트 출력
4. 테스트: 박스와 텍스트 표시

### Phase 3: 파일 목록

1. `FileSystem` - 디렉토리 읽기
2. `ListView` - 스크롤 가능한 목록
3. `FilePanel` - 파일 목록 패널
4. 테스트: 파일 목록 표시 및 스크롤

### Phase 4: 네비게이션

1. 키보드 선택 이동
2. 디렉토리 진입 (Enter)
3. 상위 디렉토리 (Backspace, ..)
4. 테스트: 디렉토리 탐색

### Phase 5: 파일 실행

1. 실행 파일 판별
2. 터미널 복원 후 실행
3. 실행 완료 후 재진입
4. 테스트: 파일 실행

### Phase 6: 완성

1. 상태바 표시
2. 기능키 바 표시
3. 색상 적용
4. 종료 처리
5. 테스트: 전체 통합

## 테스트 시나리오

### vtty 테스트

```bash
# 화면 출력 테스트
- 터미널 크기 감지
- 색상 출력 (16색, True Color)
- UTF-8 문자 출력 (한글 포함)
- 박스 문자 출력

# 입력 테스트
- 일반 문자 입력
- 화살표 키
- 기능키 (F1-F12)
- 조합키 (Ctrl, Alt)
```

### 파일 시스템 테스트

```bash
# 디렉토리 읽기
- 빈 디렉토리
- 많은 파일 (1000+)
- 숨김 파일
- 권한 없는 디렉토리

# 네비게이션
- 루트 디렉토리
- 홈 디렉토리
- 심볼릭 링크 디렉토리
```

### 통합 테스트

```bash
# 시나리오 1: 기본 탐색
1. 프로그램 시작
2. 파일 목록 확인
3. 디렉토리 진입
4. 상위 디렉토리 이동
5. 종료

# 시나리오 2: 파일 실행
1. 실행 파일 선택
2. Enter로 실행
3. 프로그램으로 복귀
```

## 예외 처리

- 권한 없는 디렉토리 접근 → 에러 메시지 표시
- 터미널 크기 변경 (SIGWINCH) → 화면 재구성
- 프로그램 인터럽트 (SIGINT) → 터미널 복원 후 종료
